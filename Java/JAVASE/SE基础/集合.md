`<>` 里面几乎什么类都能塞，但它有一个非常严格的底线法则：

**`<>` 里面只能放“引用数据类型”（也就是各种类/对象），绝对不能放“基本数据类型”！**

Java 中有 8 种基本数据类型：`int`, `double`, `boolean`, `char`, `float`, `byte`, `short`, `long`。**它们全都不准进 `<>`。**

**那如果我真的需要一个装满整数的 List 怎么办？** Java 为这 8 种基本类型提供了对应的**“包装类”（Wrapper Class）**。你必须给它们穿上“对象”的外套，才能放进泛型里：

- ❌ 错误：`List<int>` （直接报错）
    
- ✅ 正确：`List<Integer>` （把 int 包装成了 Integer 类）
    
- ❌ 错误：`List<double>`
    
- ✅ 正确：`List<Double>`
