# 返回类
## 1. **方法返回类型：

```java
// 基本类型返回
public int getAge() { ... }          // 返回int基本类型
public String getName() { ... }      // 返回String对象（Java内置类）

// 自定义类返回
public User select() { ... }         // 返回User对象（自定义类）
public List<User> getUsers() { ... } // 返回User对象的集合
```
**关键点：** Java允许方法返回任何类型的对象，包括自己定义的类

## 2.**为什么需要返回User对象？**

### 场景对比：

#### 不好的做法：返回多个独立的值
```java
// 问题：如果需要返回用户的所有信息，需要多个方法或多个返回值
public class BadUserService {
    // 需要调用4个方法才能获取完整的用户信息
    public int getUserId(String username) { ... }
    public String getUserName(String username) { ... }
    public String getRealName(String username) { ... }
    public int getUserAge(String username) { ... }
    
    // 或者使用数组/Map（但类型不安全，容易出错）
    public Object[] getUserInfo(String username) {
        return new Object[]{1, "zhangsan", "张三", 25};
        // 问题：不知道数组每个位置是什么
    }
    
    public Map<String, Object> getUserMap(String username) {
        Map<String, Object> map = new HashMap<>();
        map.put("id", 1);
        map.put("username", "zhangsan");
        map.put("name", "张三");
        map.put("age", 25);
        return map;
        // 问题：需要记住key，编译时没有类型检查
    }
}

```
#### 好的做法：返回User对象

```java
// 好处：一个方法返回所有相关信息，类型安全
public class GoodUserService {
    public User getUser(String username) {
        // 查询数据库...
        User user = new User();
        user.setId(1);
        user.setUsername("zhangsan");
        user.setName("张三");
        user.setAge(25);
        return user;  // 一个对象包含所有信息
    }
}
```
### 区别
这里只说明简单的一点:
返回一个User类的类型方法,相当于返回这个User类中所有属性的方法
