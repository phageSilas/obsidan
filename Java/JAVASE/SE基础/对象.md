
## 基本解释 
1. 对象是面向对象编程的核心，它是类的一个实例，**具有状态（属性）和行为（方法）**。
2. 对象是现实世界中事物的抽象，将现实世界中的事物映射到程序中。
3. 例如，一个“学生”对象可以有属性：学号、姓名、年龄；行为：选课、考试等。

## 面向对象与面向过程的区别
1. 面向过程：以过程（函数）为中心，强调步骤，先做什么，再做什么。数据和对数据的操作是分离的。    
2. 面向对象：以对象为中心，将数据和对数据的操作封装在一起，强调对象以及对象之间的交互。
### 举例
1. 面向过程：可能会这样设计：1. 打开洗衣机 2. 放衣服 3. 放洗衣粉 4. 清洗 5. 烘干。每一步都是一个函数，依次调用。  
2. 面向对象：**设计两个对象：人、洗衣机。人对象有方法：打开洗衣机、放衣服、放洗衣粉；洗衣机对象有方法：清洗、烘干。然后人对象调用洗衣机对象的方法来完成洗衣服。**
总结：面向对象更符合现实世界的抽象，易于维护和扩展，而面向过程则更直接，适合简单任务。

## 对象三要素
```java
// 1. 状态（State）- 属性/字段
public class Person {
    private String name;    // 状态：姓名
    private int age;        // 状态：年龄
    private String address; // 状态：地址
}

// 2. 行为（Behavior）- 方法
public class Person {
    // 状态...
    
    public void walk() {           // 行为：走路
        System.out.println("走路中...");
    }
    
    public void eat(String food) { // 行为：吃饭
        System.out.println("吃" + food);
    }
}

// 3. 标识（Identity）- 对象唯一性
public class Test {
    public static void main(String[] args) {
        Person p1 = new Person("张三", 20);  // 对象1，有自己的内存地址
        Person p2 = new Person("李四", 25);  // 对象2，不同的内存地址
        // p1和p2是不同的对象，有不同的标识
    }
}
```



## 面向对象的核心特性
### 1.封装:隐藏成员，暴露接口
```java
public class BankAccount {
    // 私有属性：外部不能直接访问
    private String accountNumber;
    private double balance;
    private String owner;
    
    // 公开方法：控制对属性的访问
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("存款成功，当前余额: " + balance);
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("取款成功，当前余额: " + balance);
        } else {
            System.out.println("取款失败，余额不足");
        }
    }
    
    // 通过getter提供有限访问
    public double getBalance() {
        return balance;
    }
    
    // 封装好处：隐藏实现细节，提高安全性
}
```
#### 补充:静态成员
静态成员（由static修饰）属于类，可以直接通过类名访问，而不需要创建对象。但是静态成员是所有对象共享的。
##### 举例
如User类中有静态属性static int age,静态方法public static void aaa(),
那么在**其他类中无需创建新对象,是可以直接调用的**:User.age=25;User.aaa();

### 2. 继承:代码复用，层次结构
```java
// 父类（基类）
public class Animal {
    private String name;
    private int age;
    
    public void eat() {
        System.out.println(name + "正在吃东西");
    }
    
    public void sleep() {
        System.out.println(name + "正在睡觉");
    }
}

// 子类（派生类）
public class Dog extends Animal {  // extends关键字表示继承
    // Dog自动拥有Animal的所有属性和方法
    
    // 可以添加新特性
    public void bark() {
        System.out.println("汪汪汪！");
    }
    
    // 可以重写（Override）父类方法
    @Override
    public void eat() {
        System.out.println("狗在啃骨头");
    }
}

public class Cat extends Animal {
    public void meow() {
        System.out.println("喵喵喵！");
    }
}
```

### 3. 多态:同一接口，不同实现
```java
// 多态示例
public class TestPolymorphism {
    public static void main(String[] args) {
        // 同一个类型（Animal）的引用，可以指向不同的子类对象
        Animal a1 = new Dog();    // 父类引用指向子类对象
        Animal a2 = new Cat();
        
        // 编译时类型是Animal，运行时类型分别是Dog和Cat
        a1.eat();  // 输出：狗在啃骨头（调用Dog的eat方法）
        a2.eat();  // 输出：...正在吃东西（调用Animal的eat方法）
        
        // 多态的好处：统一处理不同类型的对象
        Animal[] animals = {new Dog(), new Cat(), new Dog()};
        for (Animal animal : animals) {
            animal.eat();  // 每个对象会调用自己实际的eat方法
        }
    }
}

```
