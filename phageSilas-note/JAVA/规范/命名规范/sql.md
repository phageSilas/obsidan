| **类型**          | **命名风格**             | **规范描述**                                    | **示例**                                       |
| --------------- | -------------------- | ------------------------------------------- | -------------------------------------------- |
| **表名 (Table)**  | **下划线 (snake_case)** | 全小写，单词间用下划线分隔。**通常建议使用复数**（也有用单数的流派，关键是统一）。 | `users`, `order_items`, `product_categories` |
| **列名 (Column)** | **下划线 (snake_case)** | 全小写，单词间用下划线分隔。                              | `user_id`, `created_at`, `first_name`        |
| **SQL 关键字**     | **全大写 (UPPERCASE)**  | 虽然 SQL 不区分大小写，但为了区分代码和关键字，通常将关键字大写。         | `SELECT * FROM users WHERE id = 1;`          |
| **主键 (PK)**     | **下划线**              | 通常命名为 `id` 或者 `表名_id`。                      | `id` 或 `user_id`                             |
| **索引 (Index)**  | **前缀 + 下划线**         | 通常以 `idx_` 或 `uk_` (唯一索引) 开头。               | `idx_user_email`, `uk_phone_number`          |
