
# 1. 什么是 Lua？为什么要和 Redis 结合？

**Lua** 本身是一个极其轻量级、用 C 语言编写的脚本语言。它的设计初衷就是为了被嵌入到其他应用程序中，提供灵活的扩展和定制功能。

**为什么要放到 Redis 里用？（核心痛点）**

在普通的 Java 业务代码中，如果我们想执行一段逻辑，比如：“先去 Redis 里查一个值，如果等于 A，我就把它修改成 B”。

- **传统方式：** Java 程序先发一条 `GET` 命令给 Redis，拿到结果后在 Java 内存里判断，然后再发一条 `SET` 命令给 Redis。这中间需要**两次网络通信**，而且在两次通信的间隙，**极有可能被其他线程插队**（也就是破坏了原子性）。
    
- **Lua 脚本方式：** 你把这段逻辑写成一段 Lua 脚本，一次性发给 Redis。Redis 会把这段脚本**当成一个整体的命令**来执行。
    

---

# 2. Lua 脚本的核心特性（高频面试题）

做笔记的话，一定要记住 Lua 脚本在 Redis 里的三大优势：

1. ==**绝对的原子性（Atomicity）：** 这是最重要的特性。Redis 执行命令是单线程的，当 Redis 开始执行一段 Lua 脚本时，会把整个脚本作为一个整体去运行，**期间不会被任何其他客户端发来的命令插队**。这就完美替代了之前我们说的多步操作带来的并发问题。==
    
2. **减少网络开销：** 将多个业务逻辑打包在一条命令中发给 Redis，减少了客户端与服务端之间的网络往返次数（Round Trip Time, RTT）。
    
3. **可复用性：** 客户端发送的脚本会缓存在 Redis 中（通过 `SCRIPT LOAD` 和 `EVALSHA` 命令），其他客户端可以直接复用，不用每次都把一大段代码传过去。

# 3. 基础语法与核心 API

在 Redis 里写 Lua 脚本，你不需要学全套的 Lua 语法，只要记住怎么接收参数、怎么调用 Redis 命令就行了。

## 核心 API

在 Lua 脚本内部，用来执行 Redis 命令的方法主要是：

- `redis.call('命令名称', '参数1', '参数2'...)`
    

## 参数传递规则

为了规范，Redis 规定把传给脚本的参数分为两类：**键名（KEYS）** 和 **普通参数（ARGV）**。在 Lua 脚本里，它们是以数组（索引从 1 开始）的形式接收的：

- `KEYS[1]`, `KEYS[2]` ... ：代表传入的 Redis Key。
    
- `ARGV[1]`, `ARGV[2]` ... ：代表传入的具体 Value 或其他参数。
    

## 客户端执行命令格式

在 Redis 命令行中执行 Lua 脚本的标准格式是 `EVAL`：

> `EVAL "lua脚本内容" key的数量 [key1 key2...] [arg1 arg2...]`

# 4.例子

前面我们在聊互斥锁的时候，提到了一个“坑”：**释放锁的时候，必须判断锁是不是自己的，如果是才能删除。**

这一步如果用 Java 代码分两步写（先 Get 判断，再 Del），就不是原子的。我们来看看用 Lua 脚本怎么写这段**“安全释放锁”**的逻辑。

**Lua 脚本代码：**

Lua

```
-- KEYS[1] 代表锁的 Key，例如 "lock:product:1"
-- ARGV[1] 代表当前线程的标识（UUID+ThreadId）

-- 第一步：获取锁的值并比较
if redis.call('get', KEYS[1]) == ARGV[1] then
    -- 第二步：如果相等（说明是自己的锁），则执行删除
    return redis.call('del', KEYS[1])
else
    -- 第三步：如果不相等，说明锁已经过期被别人抢走了，返回 0 代表删除失败
    return 0
end
```

这段脚本一旦发给 Redis，Redis 就会瞬间一气呵成地执行完 `get`、判断 和 `del`，彻底断绝了误删别人锁的可能。

# 5. 使用 Lua 脚本的注意事项

虽然 Lua 脚本很强大，但在生产环境使用时必须小心：

- **绝对不要在 Lua 脚本里写耗时的逻辑！**
    
    因为 Redis 执行 Lua 脚本是原子性、排他性的（单线程阻塞）。如果你的 Lua 脚本里写了个死循环，或者做大量复杂的数学运算耗时过长，**整个 Redis 服务器会被彻底卡死**，无法响应其他任何客户端的请求，直接导致系统大面积瘫痪。
    
    _(Redis 默认配置 `lua-time-limit` 为 5000 毫秒，超时会报错，但这 5 秒内系统也是假死状态)_。
