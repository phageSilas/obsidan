如果说雪崩是“整片森林倒下”，那**击穿**就是“一棵参天大树被砍倒”引发的局部灾难。

---

# 1. 核心原理

缓存击穿是指：**某个极热点的 Key（比如微博热搜、秒杀商品）在过期的瞬间，同时有海量的并发请求打过来。**

因为缓存刚过期，这些请求会发现缓存没中，于是**几十万个请求同时去查数据库**并尝试回写缓存。数据库会因为瞬间负载过载而直接“罢工”。
![[image-5.png]]

---

# 2. 击穿 vs 雪崩（区别在哪？）

很多人容易混淆这两者，其实核心区别在于**规模**：

- **缓存雪崩**：大量 Key 同时到期，像洪水暴发。
    
- **缓存击穿**：**某一个**超级热点 Key 到期，像精准爆破。
    

---

# 3. 解决方案

解决击穿问题，核心思路只有两个：**要么不让它过期，要么不让请求同时钻过去。**

## A. 设置热点数据“永不过期”(逻辑过期)

从物理层面上，我们不给这个热点 Key 设置过期时间（TTL）。

- **逻辑过期：** 在 Value 内部存一个“过期时间”字段。
    
- **后台更新：** 由后台异步线程定时去检查并更新这个 Key。
    
- **优点：** 性能最高，请求永远不会被阻塞。
    
- **缺点：** 可能会存在短暂的数据不一致（后台还没更新完，用户看到的是旧数据）。

## B. 使用互斥锁 (Mutex Lock)

这是最常用的硬核方案。当缓存失效时，不让所有请求都去查快数据库，而是只允许**第一个**抢到锁的请求去查 DB，其他请求**等待或重试**。

### 关键语法 setnx
SETNX 是『SET if Not eXists』(如果不存在，则 SET)的简写。

> SETNX key value  
> 将 key 的值设为 value ，当且仅当 key 不存在。  
> 若给定的 key 已经存在，则 SETNX 不做任何动作。

 **语法:** SETNX key value

- **key**: 要设置的键。
    
- **value**: 要设置的值。

**返回值**

- 设置成功，返回 _1_。
    
- 设置失败（键已存在），返回 _0_。
    

示例

``` 
 EXISTS job # 检查键是否存在

(integer) 0 # job 不存在

  

 SETNX job "programmer" # 设置键 job 的值为 "programmer"

(integer) 1 # 设置成功

  

 SETNX job "code-farmer" # 尝试覆盖键 job 的值

(integer) 0 # 设置失败，因为 job 已存在

   

 GET job # 获取键 job 的值

"programmer" # 值未被覆盖

```

### 使用缓存空值解决缓存穿透以及使用互斥锁解决缓存击穿的例子
``` java
// 获取锁 (修改：存入线程唯一标识，防止误删)
    public boolean setlock(String lockKey, String threadId) {
        // 将当前线程的 ID 或 UUID 作为 Value 存入
        Boolean flag = stringRedisTemplate.opsForValue().setIfAbsent(lockKey, threadId, 10, TimeUnit.SECONDS);
        return BooleanUtil.isTrue(flag);
    }

    // 释放锁 (修改：判断是自己的锁才删，避免误删别人的锁)
    public void unLock(String lockKey, String threadId) {
        String currentLockVal = stringRedisTemplate.opsForValue().get(lockKey);
        // 判断当前 Redis 里的锁是不是自己加的
        if (threadId.equals(currentLockVal)) {
            stringRedisTemplate.delete(lockKey);
            // 注意：生产环境这里推荐用 Lua 脚本保证原子性，这里为了好理解用代码写了。
        }
    }

    @Override
    public Result queryShopById(Long id) {
        String key = RedisConstants.CACHE_SHOP_KEY + id;
        
        // 1. 从redis查询店铺缓存
        String shopJson = stringRedisTemplate.opsForValue().get(key);
        
        // 2. 缓存命中真实数据
        if (StrUtil.isNotBlank(shopJson)) {
            Shop shop = JSONUtil.toBean(shopJson, Shop.class);
            return Result.ok(shop);
        }
        
        // 3. 缓存命中"空值" (防穿透)
        if (shopJson != null) {
            return Result.fail("店铺不存在");
        }

        // 4. 缓存不存在，准备获取互斥锁查数据库
        String lockKey = RedisConstants.LOCK_SHOP_KEY + id;
        String threadId = UUID.randomUUID().toString(); // 生成当前线程的锁标识
        
        try {
            boolean isLock = setlock(lockKey, threadId);

            // 4.1 没有抢到锁，进行重试 (防击穿)
            if (!isLock) {
                Thread.sleep(50);
                return queryShopById(id); // 如果想更安全，最好加个最大重试次数限制
            }

            // ==========================================
            // 🚨 4.2 【关键修复】：获取锁成功后，Double Check (双重检查)
            // ==========================================
            shopJson = stringRedisTemplate.opsForValue().get(key);
            if (StrUtil.isNotBlank(shopJson)) {
                return Result.ok(JSONUtil.toBean(shopJson, Shop.class));
            }
            if (shopJson != null) {
                return Result.fail("店铺不存在");
            }

            // 4.3 确认缓存真的没有，查数据库
            Shop shop = getById(id);
            log.debug("从数据库查询店铺信息{}", shop);

            // 5. 数据库不存在，写入空值 (防穿透)
            if (shop == null) {
                stringRedisTemplate.opsForValue().set(key, "", RedisConstants.CACHE_NULL_TTL, TimeUnit.MINUTES);
                return Result.fail("店铺不存在");
            }

            // 6. 存在，写入redis
            stringRedisTemplate.opsForValue().set(key, JSONUtil.toJsonStr(shop), RedisConstants.CACHE_SHOP_TTL, TimeUnit.MINUTES);
            return Result.ok(shop);

        } catch (Exception e) {
            throw new RuntimeException(e);
        } finally {
            // 4.4 释放锁 (传入标识，只删自己的锁)
            unLock(lockKey, threadId);
        }
    }

```
