## 解决办法：
以上这些都是lombok插件的@Data失效导致的，解决如下：
1) 修改pom.xml配置文件,将spring自带的限制插件注释掉
![[Attachment/image-3.png]]
2) 去掉@Data,手动添加get/set方法



#  nginx打不开:

# 即使置文件中配置了密钥,id等各种配置数据,但运行后查询到的仍为null

1. 看看yml文件中的key: 和Value中间是否加了空格(雷霆问题)
2. 看看yml中的value的引用值名称是否和dev.yml文件中的key值一样

# 使用@Test测试时,报错NoSuchMethodError

原因是junit和spring版本不兼容,通过依赖关系查看项目创建时自带的junit版本,并在pom文件中修改版本号,(主要是junit-jupiter-api)
