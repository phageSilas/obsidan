# 4. 自动拆箱导致空指针异常 `NullPointerException` 
##  Java 中的装箱（Boxing）与拆箱（Unboxing）

Java 是一种面向对象的语言，但为了保证运行效率，保留了 8 种**基本数据类型**（如 `int`, `boolean`, `double` 等）。然而，在很多场景下（例如向 `List`, `Map` 等泛型集合中存入数据），Java 要求必须使用**对象**。

为了解决这个问题，Java 为每种基本类型提供了对应的**包装类**（Wrapper Classes，如 `Integer`, `Boolean`, `Double` 等）。

### 1. 什么是自动装箱与拆箱？

从 Java 5 开始，编译器引入了自动装箱和拆箱机制，让开发者可以无缝地在基本类型和包装类之间切换，不需要再手动去 `new` 对象。

- **自动装箱 (Autoboxing)**：将基本数据类型自动转换为对应的包装类对象。
    
    - **代码**：`Integer a = 10;`
        
    - **底层原理**：编译器会在编译时自动将代码转换为 `Integer a = Integer.valueOf(10);`。
        
- **自动拆箱 (Unboxing)**：将包装类对象自动转换为对应的基本数据类型。
    
    - **代码**：`int b = a;`
        
    - **底层原理**：编译器会自动将代码转换为 `int b = a.intValue();`。
        

### 2. 自动拆箱的陷阱：空指针异常 (NPE)

自动拆箱极大地简化了代码，但也埋下了一个巨大的地雷：**如果包装类对象为 `null`，在自动拆箱时就会抛出 `NullPointerException` (空指针异常)。**

Java

```
Integer a = null;
// 下面这行代码会报错，因为相当于执行了 a.intValue()，而 a 是 null
int b = a; 

Boolean flag = null;
// 下面这行代码会报错，因为 if() 内部需要一个基本类型的 boolean
// 编译器会执行 flag.booleanValue()，由于 flag 为 null，直接抛出 NPE
if (flag) { 
    System.out.println("执行了");
}
```
