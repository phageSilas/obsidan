<>中的内容本质和[[关于方法返回类型的探究]]中的情况差不多
集合的本质：**集合是用来装对象的容器。**

在真实的软件开发中，我们要处理的数据可不只有简单的文本（`String`）。 假设你正在开发一个公司的人事系统，你需要把公司里所有的“部门”列出来，装在一个列表里传给前端展示。 这里的“部门”在代码中怎么表示？就是你自己写的一个 `Dept` 类！

- `List<String>`：一箱子文本。
    
- `List<Dept>`：一箱子“部门对象”。每个 `Dept` 对象里可能包含了“部门编号”、“部门名称”、“部门经理名字”等复杂信息。
    

**只要是一个合法的“类（Class）”，不管是 Java 自带的（如 String），还是你自己手写的（如 Dept, User, Order），都可以无缝塞进 `<>` 里。**
