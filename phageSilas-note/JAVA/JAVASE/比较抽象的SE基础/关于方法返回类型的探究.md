可以先看[[自定义返回类型的方法]]
对于一个方法,其基本创建结构是
``` java
(权限修饰词) 返回类型 方法名(参数){
 方法体;
 }
```

1) 对于常见的int,String之类的,举一个简单的例子
``` java
public int sum(a,b){
return a+b;
}
```
表明其他地方调用sum(a,b)方法时,需要用到a+b的值,然后这个值是int类型,那么
int sum(a，b)等效于int sum=a+b,方法体中的return就是把经由方法体的某些步骤得出的int类型的数据返回给sum.即相当于把这个数据赋值给sum(a，b)

2) 当返回类型为void时,表明本方法无需返回值,执行完方法体内的步骤就结束了,不用把这些步骤得到的什么结果返回给"方法名"(也就是其他方法不需要本方法的结果,只需要在调用本方法时,本方法执行完所有步骤就行),即不需要return

3) 对于一些奇怪的返回类型,比如类和集合或者其他抽象玩意,说白了和第一点差不多,不过第一点是一个个单独的属性,比如一个sum()对应一个int值;
	类或者集合相当于一个大包,然后大包里有一堆属性, **这种返回类型的方法常见于给修改一个整体中的好几个属性** ,比如同时修改部门一中的部门名称,部门id之类的属性
``` java
	public Dept abc(){
	Dept dept = new Dept();
	dept.setName("研发部");
	dept.setDeptId(3);

	return dept;
	}
```
假设A方法调用该abc方法,**表明A方法也需要同时获取或者修改一个部门**,那么如果setName和setDeptId是独立的方法,那A方法就要调用两个方法,消耗性能,但abc相当于一个类,abc中同时有name,deptId两个属性,调用abc就相当于开箱子A-->abc-->name+deptId,一个方法就包括了多种参数,无需调用多个方法;(集合同理)
