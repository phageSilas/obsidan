# 1.使用简要
一般来说,接口只定义一个规范规则,就是只定义一些共用的方法,**只有方法名,没有方法体**,其他类调用该接口后,可以根据自己类的需求**自行重写方法,这些重写的方法互相独立**,互不影响(除非是继承这些类的子类)。
总之,接口类似于产品制作标准,其他类可以根据这个标准自行修改方法
# 2.基本定义
接口是一种引用类型，它是抽象方法的集合。接口还可以包含常量（默认为public static final）。接口的主要目的是定义一组规范，让不同的类去实现这些规范，从而实现多态。

# 3.特点
1. 接口中的方法默认为public abstract（抽象方法），不能有方法体（Java 8之前）。从Java 8开始，接口可以包含默认方法（default method）和静态方法（static method），这两种方法可以有方法体。
2. 接口中的变量默认为public static final，即常量。
3. 接口不能实例化，即不能创建接口的对象。
4. 类通过implements关键字实现接口，并且必须实现接口中所有的抽象方法（除非该类是抽象类）。
5. 一个类可以实现多个接口，从而实现了多重继承的效果。

# 4.通俗示例
```java
// 接口 = 商品目录（只告诉你有什么功能）
interface 家电 {
    void 开机();
    void 关机();
    default void 清洁() {  // 可选服务
        System.out.println("基础清洁");
    }
}

// 实现类 = 具体的商品（各自独立生产）
class 电视 implements 家电 {
    @Override
    public void 开机() {
        System.out.println("电视开机，显示Logo");
    }
    
    @Override
    public void 关机() {
        System.out.println("电视关机，显示待机灯");
    }
    
    @Override
    public void 清洁() {
        System.out.println("用屏幕清洁剂擦拭");
    }
}

class 空调 implements 家电 {
    @Override
    public void 开机() {
        System.out.println("空调开机，开始送风");
    }
    
    @Override
    public void 关机() {
        System.out.println("空调关机，关闭压缩机");
    }
    // 不重写清洁()，使用默认的基础清洁
}
```
# 5.基础代码示例
```java
class Animal {
    public void makeSound() {
        System.out.println("动物发出声音");
    }
}

class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("汪汪！");  // 重写只在Dog及其子类中生效
    }
}

class GoldenRetriever extends Dog {
    // 继承Dog的重写版本
    // 可以再次重写
    @Override
    public void makeSound() {
        System.out.println("温柔的汪汪！");
    }
}

class Cat extends Animal {
    // 不受Dog重写的影响
    // 使用Animal的原始版本
    // 也可以自己重写
    @Override
    public void makeSound() {
        System.out.println("喵喵！");
    }
}
```
