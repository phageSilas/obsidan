## 区别
无参构造和有参构造可以简单理解为:
有参构造已经初始化了所有的属性,调用时直接在括号中一一对应填写即可
无参构造没有初始化对象,仍为为默认值,调用时需要使用set方法

### 1. **有参构造方法（Parameterized Constructor）**

```java
public User(Integer id, String username, String password, String name, Integer age) {
    this.id = id;           // 将参数赋值给当前对象的id属性
    this.username = username; // 将参数赋值给当前对象的username属性
    this.name = name;       // 将参数赋值给当前对象的name属性
    this.age = age;         // 将参数赋值给当前对象的age属性
}
```

**作用：**

- **创建对象时直接初始化所有属性**
    
- 一次性给对象的5个属性赋值
    
- 调用方式：`User user = new User(1, "张三", "123", "张三丰", 25);`
    

### 2. **无参构造方法（No-argument Constructor）**

```java
public User() {
    // 空方法体，只是创建对象，属性保持默认值
    // id=null, username=null, name=null, age=null等
}
```

**作用：**

- **只创建对象，不初始化属性**
    
- 调用方式：`User user = new User();`
    
- 属性保持默认值（null、0等）

### 3. **无参构造为什么也能设置属性？**

```java
// 示例：通过无参构造 + Setter设置属性
@Test
public void testUser() {
    // 1. 使用无参构造创建对象（属性都是null/默认值）
    User user = new User();
    
    // 2. 通过Setter方法逐个设置属性值
    user.setId(1);                    // 调用setId方法
    user.setUsername("zhangsan");     // 调用setUsername方法
    user.setPassword("123456");       // 调用setPassword方法
    user.setName("张三");             // 调用setName方法
    user.setAge(25);                  // 调用setAge方法
    
    // 3. 通过Getter方法获取属性值
    System.out.println(user.getId());     // 输出：1
    System.out.println(user.getName());   // 输出：张三
}
```

### 4. **实际应用场景**

**有参构造适合：**

```java
// 场景1：创建时就知道所有属性值
User user1 = new User(1, "admin", "123", "管理员", 30);

// 场景2：快速创建测试数据
List<User> users = Arrays.asList(
    new User(1, "user1", "pass1", "用户1", 20),
    new User(2, "user2", "pass2", "用户2", 25)
);

```
**无参构造 + Setter适合：**

```java
// 场景1：从表单/JSON反序列化对象
User user = new User();
user.setId(json.getInt("id"));
user.setUsername(json.getString("username"));
// ... 逐个设置

// 场景2：MyBatis等框架反射创建对象
// MyBatis会：1.调用无参构造 2.通过Setter设置查询结果
// 这要求必须有无参构造和Setter

// 场景3：Builder模式/链式调用
User user = new User()
    .setId(1)
    .setUsername("test")
    .setName("测试");
```
