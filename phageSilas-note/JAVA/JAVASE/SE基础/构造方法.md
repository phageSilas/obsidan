
## 基本解释
1. 构造方法是一种特殊的方法，用于创建对象时初始化对象。
2. **构造方法的名字必须与类名完全相同，并且没有返回类型（连void都没有）。**
3. 当使用new关键字创建对象时，会自动调用构造方法。
4. 构造方法可以重载，即可以有多个参数不同的构造方法。
5. 分为有参构造和无参构造,如果没有显式定义构造方法，Java会提供一个默认的无参构造方法（如果已经定义了有参构造，则不会自动生成无参构造，除非显式定义）

```java
public class User {
    private String name;
    private int age;
    
    // 1. 构造方法名必须与类名完全相同
    // 2. 没有返回类型（连void都没有）
    // 3. 可以重载（多个构造方法）
    
    // 无参构造方法
    public User() {
        System.out.println("无参构造被调用");
        this.name = "默认姓名";
        this.age = 0;
    }
    
    // 有参构造方法
    public User(String name, int age) {
        System.out.println("有参构造被调用");
        this.name = name;
        this.age = age;
    }
    
    // 普通方法
    public void printInfo() {
        System.out.println(name + ", " + age);
    }
}
```
## 和普通方法的区别
1. **构造方法的方法名必须与类名相同**，普通方法则没有这个限制。
2. **构造方法没有返回类型，普通方法必须有返回类型（包括void）**。
3. 构造方法是在创建对象时由new关键字自动调用的，普通方法需要通过对象或类名（静态方法）来调用。
4. 构造方法的作用是初始化对象，而普通方法的作用是执行特定的操作。

```java
public class ConstructorVsMethod {
    // 构造方法示例
    public ConstructorVsMethod() {    // 1. 名=类名
        // 初始化代码                        // 2. 无返回类型
    }                                     // 3. 创建对象时自动调用
    
    // 普通方法示例
    public void doSomething() {        // 1. 名任意（符合规范）
        // 业务逻辑代码                     // 2. 必须有返回类型
    }                                     // 3. 需要显式调用
    
    public int calculate() {           // 返回int类型
        return 10;
    }
    
    public static void staticMethod() { // 静态方法
        System.out.println("静态方法");
    }
}
```
